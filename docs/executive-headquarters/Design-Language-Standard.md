> When it conflicts with a requested change, stop and flag the conflict before proceeding.

---

## What Headquarters is not

- It is **not** a SaaS application.
- It is **not** a documentation site.
- It is **not** an admin dashboard.
- It is **not** a component gallery.

## What Headquarters is

**It is a place.**

Every interface should feel like entering a beautifully designed residence where work happens naturally. The user should feel like they are **inhabiting a space, not operating software.**

---

## Design philosophy — the references

- **Apple-level craftsmanship** and attention to detail.
- The **architectural warmth** of Aman, Four Seasons, Edition Hotels, or a luxury private residence.
- The **editorial refinement** of premium magazines and beautifully designed books.
- The **exploration and delight** of Apple's Design Resources — *not* a documentation website.
- The **clarity of Linear** only where it genuinely improves usability, **not as a visual identity.**

---

## Priorities — in this order, always

1. **atmosphere** before widgets
2. **space** before density
3. **materials** before decoration
4. **light** before gradients
5. **restraint** before features
6. **calm** before productivity

---

## How it should feel

- **Navigation** should feel like **moving through architecture.**
- **Cards** should feel like **objects placed intentionally within a room.**
- **Motion** should feel **physical.**
- **Typography** should feel **editorial.**
- **Nothing** should feel like a template.

---

## The tie-breaker

When multiple technically correct solutions exist, **choose the one that creates the strongest emotional sense of place.**

---

## How this governs future work

- **Precedence.** This standard governs every Headquarters and Studio Mode surface. Studio Mode is a layer *within* Headquarters and inherits this language in full — the design lab evaluates options *against* this standard, never outside it.
- **Default posture.** Reach for a "default SaaS convention" (a dashboard grid, a status badge, a progress bar, a settings panel, a red notification dot, a generic card wall) **only** when it genuinely improves the experience — and prefer the residence-native expression first. Retire corporate/dashboard vocabulary in UI and copy (see Experience-Architecture.md · *A Note on Language*).
- **Applied to a sprint, this means:** lead with the room, the light, the material, and the negative space; add the interface quietly inside it; earn every element; and measure the result by whether it deepens the sense of place — not by feature count or density.
- **On conflict.** If a Founder request would contradict this standard, **surface the conflict and ask before overriding.** It is a permanent canon (see [institutional standards protocol]); it changes only by the Founder explicitly superseding it.
